# Skill: QA and Test Strategy

## Tujuan

Membuat fitur lolos uji bisnis, bukan sekadar lolos compile.

## Fokus testing

- workflow status
- role permissions
- perhitungan nominal dan fee
- laporan harian
- responsive behavior desktop/tablet/mobile
- edge case pending/gagal/refund

## Level test

- unit test untuk service/action
- feature test untuk endpoint dan policy
- browser/manual QA untuk flow penting

## Acceptance minimum

- happy path
- invalid input
- unauthorized access
- status illegal transition
- viewport kecil
