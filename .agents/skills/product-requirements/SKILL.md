# Skill: Product Requirements

## Tujuan

Ubah permintaan bisnis menjadi scope implementasi yang jelas, terukur, dan bisa dipecah ke modul.

## Gunakan saat

- ada fitur baru
- ada perubahan workflow bisnis
- ada request dari owner yang masih umum

## Langkah kerja

1. Identifikasi aktor: owner, admin, kasir, operator, supervisor.
2. Pisahkan proses barang fisik vs transaksi digital.
3. Tulis tujuan bisnis, risiko, input, output, dan definisi selesai.
4. Tentukan apakah fitur masuk MVP, fase 2, atau backlog.
5. Catat dependency ke role, kas, laporan, audit, dan responsive UI.

## Output wajib

- ringkasan tujuan
- user flow singkat
- acceptance criteria
- data yang diperlukan
- dampak ke modul lain

## Larangan

- jangan lompat ke coding sebelum scope dan status bisnis jelas
- jangan campur requirement web dan mobile bila prioritasnya berbeda
